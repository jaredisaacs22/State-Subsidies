Swaps: none yet. Proposed swaps are filed in `CHANGELOG.md` with a justification; they take effect only after COO sign-off.

## Platforms we considered and dropped, with reasons

| Considered | Dropped because | Could be reinstated for |
|---|---|---|
| **Bloomberg Terminal** | Power-user bar; wrong audience ceiling for a public good | Data-layer ops references only (never UX) |
| **GrantWatch / GrantStation** | Direct competitors; not world-class; benchmarking them targets mediocrity | Used as **floors** in SS-007 and SS-003 — "must be at least this good" |
| **Airbnb** | Trust-shape is subtly different; not a directory of money; risk of mis-borrowing "hospitality warmth" where we need "fiduciary warmth" | Occasional motion/empty-state inspiration only |
| **Glassdoor** | Trust model is crowd-sourced; incompatible with our provenance model | Never |
| **Indeed** | Commercial jobs directory; the UX ethic is aggressively monetized | Never |

## How each item uses this set

Each item's Section 5 is a **matrix**, not a list. For each benchmark that applies, the matrix states:

- What they do on *this specific dimension*.
- What we borrow.
- What we consciously avoid.

No item is permitted to cite a benchmark without filling all three columns. Otherwise the benchmark is decoration; decoration rots.
